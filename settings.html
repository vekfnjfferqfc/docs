Build settings Configure the template via template.config.js

This page explains the main options inside template.config.js. The idea is simple: you can enable/disable features and adapt the workflow without touching build code.

Where to edit settings

Open template.config.js in the project root. This file is the “control panel” of the build system: server, assets pipeline, styles, scripts, and optional modules.

Tip
Change one option at a time and run npm run dev to confirm the result.

Editor helpers

vscode

Settings for VS Code integration (auto-config of helpful things).

  • settings — auto setup of path aliases for Path Autocomplete.
  • snippets — auto add project snippets (can be disabled).

Dev tools: components page & page generator

devcomponents

Controls the “components playground” page used during development.

  • enable — enable/disable the page (true/false).
  • filename — filename of the playground page.

newpage

Controls the “create new page” feature (generator).

  • copyfromindex — create a page based on src/index.html.
  • usetemplate — which template to use when copyfromindex: false.

GitHub publishing & navigation panel

git

Settings used by the GitHub publish command.

  • repo — repository path.
  • branch — branch name.

navpanel

Controls an optional navigation panel for quick access during development / build previews.

  • dev — show in dev mode (true/false).
  • build — show in production output (true/false).
  • position — left / right.
  • color — text/links color.
  • background — background color.
  • transition — open/close animation speed (ms).

Project statistics

statistics

Controls whether the build system collects and prints project statistics.

  • enable — allow statistics collection (true/false).
  • showonbuild — auto print stats after production build (true/false).

Local server & output behavior

server

Controls the local dev server and how output paths are generated.

  • path — paths format: ./ relative or / absolute.
  • isassets — put final CSS/JS into dist/assets (true/false).
  • buildforlocal — build to open from file explorer (no server). Forces single JS/CSS bundle.
  • copyfiles — copy src/files to dist/files on build (true/false).
  • version — add cache-busting version to CSS/JS (true/false).
  • hostname — server host name.
  • port — server port.
Note
If you enable buildforlocal, scripts and styles are forced into one bundle so the site can work without a server.

HTML processing

html

Controls how HTML is formatted in production builds.

  • beautify.enable — enable formatting (true/false).
  • beautify.indent — indentation style.

Styles & SCSS pipeline

styles

Controls CSS/SCSS compilation and production optimizations.

  • tailwindcss — enable Tailwind (true/false).
  • pxtorem — convert px → rem on build (true/false).
  • critical — generate critical CSS and inject into HTML (true/false).
  • codesplit — split CSS by pages and load only what is needed (true/false).
  • devfiles — create unminified copies in production (true/false).

Fonts

fonts

Controls fonts processing and optional automation.

  • iconsfont — generate icon font from SVG icons (true/false).
  • download — download remote fonts (e.g. Google Fonts) and make them local (true/false).

Images pipeline

images

Controls image conversion, resizing, and optimization in production builds.

  • svgsprite — generate SVG sprite (true/false).
  • optimize.enable — enable optimization (true/false).
  • optimize.edithtml — rewrite HTML to match generated formats/sizes.
  • optimize.sizes — extra image widths to generate.
  • optimize.dpi — multipliers for extra files (e.g. 2x).
  • optimize.attrignore — if an <img> has this attribute, skip processing.
  • modernformat.enable — convert to modern formats (true/false).
  • modernformat.type — webp / avif.
  • modernformat.only — keep only modern files after convert (true/false).
  • modernformat.quality — quality for modern formats.
  • jpeg.quality — JPEG optimization quality.
  • png.quality — PNG optimization quality.
If optimization is disabled
Images will be copied “as is” (no conversion and no compression).

JavaScript pipeline

js

Controls component connection, production output, and framework integration.

  • hotmodules — auto-detect and connect components (true/false).
  • devfiles — create unminified copies in production (true/false).
  • bundle.enable — force one JS + one CSS output file (true/false).
  • react — enable React processing (true/false).
  • vue — enable Vue processing (true/false).

PHP server & PUG

php

Local PHP server settings (only if you use it).

  • enable — enable PHP server (true/false).
  • base — folder where PHP files live.
  • hostname — host name.
  • port — port.
  • binary — path to PHP interpreter.
  • ini — path to php.ini.

pug

Enables the PUG preprocessor.

  • enable — enable PUG (true/false).

FTP deploy (optional)

ftp

Used by the production deploy command to upload files from dist/ to a hosting server.

  • host — FTP host.
  • port — FTP port.
  • remoteDir — remote folder.
  • user — username.
  • password — password.
Security
Don’t commit real FTP passwords to public repositories. Use environment variables if needed.

Logs

logger

Controls logging in the terminal and browser console.

  • terminal — module logs in terminal (true/false).
  • console.enable — component logs in browser console (true/false).
  • console.removeonbuild — remove logs in production builds (true/false).

Project page generator

projectpage

Generates a “project page” in production with links to all HTML pages — useful for demos.

  • enable — enable generator (true/false).
  • projectname — title text (empty → uses folder name).
  • template — template file path.
  • outfilename — output file name (empty → uses folder name).

Path aliases

aliases

Controls path aliases used in the project (for example in HTML/JS/SCSS imports). Aliases help you avoid long relative paths.

Break reminder

coffee

Shows a periodic reminder message in the terminal.

  • enable — enable reminder (true/false).
  • text — message text.
  • interval — interval in minutes.
